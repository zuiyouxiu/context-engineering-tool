# 上下文工程管理工具 v2.0 - 配套提示词

## 🎯 核心使用提示词

### 会话开始时的基础提示词

```
我正在使用上下文工程管理工具 v2.0 来管理项目上下文。请在每次会话开始时执行以下操作：

1. 使用 `get-context-info` 工具获取完整的项目上下文信息，包括：
   - 项目核心文档（productContext.md, activeContext.md, progress.md, decisionLog.md, systemPatterns.md）
   - 我的用户偏好和学习进度
   - 最近的对话历史

2. 基于获取的上下文信息，了解：
   - 项目的当前状态和目标
   - 我的编程风格和偏好
   - 最近的变更和关注点

请现在就执行这个操作，获取项目上下文。
```

### 任务执行时的智能上下文构建提示词

```
在处理具体任务时，请使用 `build-dynamic-context` 工具为当前任务构建智能上下文：

参数设置：
- taskType: 根据任务性质选择（architecture/feature/bugfix/refactor/decision/progress/general）
- userInput: 我的具体需求描述
- priority: 任务优先级（high/medium/low）
- sessionId: 当前会话标识符

这个工具会：
- 自动整合多源信息（项目文件、记忆、搜索结果）
- 评估上下文质量，确保你能完成任务
- 提供优化建议

请根据我的任务类型使用这个工具。
```

### 需要外部信息时的知识检索提示词

```
当需要获取技术信息、最新文档或相关示例时，请使用 `retrieve-external-knowledge` 工具：

参数设置：
- query: 搜索查询内容
- taskType: 任务类型
- sources: 选择信息源 ['web', 'code', 'files', 'libraries']
  - web: 网络搜索获取最新信息
  - code: 在项目代码中搜索相关实现
  - files: 在项目文件中搜索相关内容
  - libraries: 查询第三方库文档

这能帮助获取最新、最相关的技术信息来辅助任务完成。
```

### 完成重要工作后的更新提示词

```
在完成重要的架构变更、功能开发、问题修复等工作后，请使用 `update-context-engineering` 工具获取文档更新指导：

参数设置：
- changeType: 变更类型（architecture/feature/bugfix/refactor/decision/progress）
- description: 变更的简要描述

然后按照工具返回的详细指导更新相关的上下文工程管理文件，确保项目文档与实际状态保持一致。
```

## 📊 质量保证提示词

### 上下文质量检查

```
在处理复杂任务时，请特别关注 `build-dynamic-context` 工具返回的质量评估信息：

- 完整性评分：低于80分时需要补充信息
- 可行性评估：确保当前上下文足以完成任务
- 优化建议：按照建议改进上下文质量

如果质量评分过低，请主动获取更多信息或澄清需求。
```

### 任务完成度验证

```
任务完成后，请验证：

1. 是否达到了预期目标
2. 是否更新了相关文档
3. 是否记录了重要决策和经验
4. 是否需要更新用户偏好和学习进度

确保每次协作都能积累经验，提升后续效率。
```

## 🔄 工作流程提示词

### 标准工作流程

```
请按照以下标准流程与我协作：

1. **会话开始**：使用 `get-context-info` 获取项目上下文
2. **任务分析**：理解需求，确定任务类型和优先级
3. **上下文构建**：使用 `build-dynamic-context` 构建任务上下文
4. **信息获取**：必要时使用 `retrieve-external-knowledge` 获取外部信息
5. **任务执行**：基于完整上下文执行任务
6. **记录更新**：使用相应工具更新文档和记忆
7. **质量检查**：验证任务完成度和上下文更新

这样确保每次协作都是结构化和高质量的。
```

### 持续改进流程

```
在我们的长期协作中，请定期：

1. 分析我的偏好变化和学习进度
2. 优化上下文构建策略
3. 调整信息获取和格式化方式
4. 完善工作流程

让我们的协作越来越高效和个性化。
```

3. **记忆学习**：持续优化我们的协作方式
4. **多源整合**：利用各种信息源获得最佳结果
